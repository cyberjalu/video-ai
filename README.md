# ClipNews AI — Tauri Desktop App

> **Turn any article into a TikTok-ready video.**

A Tauri desktop app that converts news article URLs into polished vertical MP4 videos with AI-generated narration, captions, and scene layout.

---

## Features

- Paste any news article URL
- AI extracts content, captures screenshots, writes script, generates TTS voiceover
- Renders a 1080×1920 vertical TikTok-style MP4
- Clean dark-mode desktop UI (Linear/Raycast aesthetic)
- Real-time generation progress tracking
- Scene plan viewer, article metadata preview
- Video preview and download

---

## Tech Stack

- **Frontend**: React + TypeScript + Tailwind CSS v4 + Framer Motion
- **Desktop shell**: Tauri v2
- **Rendering pipeline**: Remotion + Playwright + Google Gemini API
- **Icons**: lucide-react
- **Font**: Montserrat via @fontsource

## Getting Started

### Prerequisites

- Node.js 18+
- Rust + Tauri CLI (`cargo install tauri-cli` or `npm install -D @tauri-apps/cli`)
- Gemini API key (get one at [Google AI Studio](https://aistudio.google.com))

### Install

```bash
npm install
```

### Run in Development (with Tauri)

```bash
npm run tauri dev
```

### Run Frontend Only (for UI dev without Rust)

First, install `@tailwindcss/vite` for proper Tailwind v4 support:

```bash
npm install -D @tailwindcss/vite
```

Or pre-generate the Tailwind CSS and use it directly:

```bash
npm run generate:css   # generates src/tailwind-generated.css
npm run dev            # starts Vite dev server on port 1420
```

> **Note**: The `tailwind-generated.css` file is auto-generated and gitignored. Run `npm run generate:css` after any significant class changes.

### Build

```bash
npm run build          # regenerates CSS + TypeScript check + Vite build
npm run tauri build    # full desktop app build
```

## How It Works

### UI State Machine

The frontend maps backend worker events to friendly UI states:

| Backend `WorkerStep` | UI Status | User-friendly label |
|---|---|---|
| `extract` | `reading_article` | Reading article |
| `screenshot` | `capturing_screenshots` | Capturing screenshots |
| `plan`, `plan_rewrite` | `writing_script` | Writing video script |
| `tts`, `audio_fit` | `generating_voiceover` | Generating voiceover |
| `render` | `rendering_video` | Rendering video |
| `qc` | `finalizing_export` | Finalizing export |

The backend emits JSON events on each line of `render_log`:

```ts
type WorkerEvent =
  | { type: "step_start"; step: WorkerStep }
  | { type: "step_done"; step: WorkerStep; plan?: VideoPlan }
  | { type: "log"; message: string }
  | { type: "done"; projectDir: string; mp4: string }
  | { type: "error"; message: string };
```

### CSS Setup (Tailwind v4)

This project uses Tailwind CSS v4. Since `@tailwindcss/vite` is a separate package, CSS is pre-generated by `generate-tailwind.cjs`:

```bash
node generate-tailwind.cjs
# → Generates src/tailwind-generated.css (~32KB)
```

For best DX with live-reloading utilities, install:

```bash
npm install -D @tailwindcss/vite
```

Then update `vite.config.ts`:

```ts
import tailwindcss from "@tailwindcss/vite";

export default defineConfig({
  plugins: [react(), tailwindcss()],
  // ...
});
```

And update `src/styles.css` to:

```css
@import "tailwindcss";
```

## Settings

Settings are stored in `localStorage`:

| Key | Description |
|---|---|
| `GEMINI_API_KEY` | Your Gemini API key (required for TTS + script) |
| `CLIPNEWS_LICENSE_KEY` | License key (future activation flow) |
| `RENDER_PRESET` | `deep_explainer` / `news_60_80` / `ultra_25_35` |
| `RENDER_LAYOUT_MODE` | `tri` / `dual` / `mono` |
| `RENDER_ENABLE_CALLOUTS` | `true` / `false` |
| `RENDER_ENABLE_PROGRESS` | `true` / `false` |
